# Amia-plugin-welcome

群成员加入与离开提示插件。

## 项目定位

插件监听 OneBot V11 的 `GroupIncreaseNoticeEvent` 和 `GroupDecreaseNoticeEvent`，查询成员显示名，生成欢迎或离开消息，并发送到对应群。它不负责 qbind、入群审批、群权限或完整群规帮助。

## 当前已实现

- 监听成员加入和成员减少通知；
- 优先读取群成员名片，其次使用昵称，查询失败时回退到 user_id；
- 使用 QQ 头像 URL 作为图片段；
- 入群消息按 `@成员 + 文本 + 图片` 组合；
- 离群消息按 `文本 + 图片` 组合；
- 头像或成员资料查询失败时继续使用回退值。

当前代码没有机器人自身加入/离开跳过逻辑，没有 Markdown 构建，也没有群级开关和模板配置；README 不把这些能力描述为现状。

## 尚未验证

以下兼容性仍需结合当前部署的 Gensokyo 版本确认：

- 实际 Gensokyo 入群事件；
- 实际 Gensokyo 离群事件；
- 原始 `CQ:member` 是否必须保留；
- `CQ:member`、`CQ:at`、`CQ:markdown` 的实际顺序要求；
- 机器人自身加入或离开时是否应跳过；
- 离群事件不能据此伪造“被踢”判断；
- 头像获取失败时的发送降级；
- Markdown 构建失败时的发送降级。

当前 `qqbot/config.yml` 只有 Gensokyo 配置版本 `1`，本地可执行文件没有可读的 Release006/007/008 版本元数据。因此不能仅凭工作目录确认部署版本，必须从实际启动日志、发布包或运行中的版本接口核对。

## 测试计划

先执行离线检查：

```powershell
python -m compileall -q .
python -m unittest discover -s tests -v
```

然后用 NoneBot 模拟事件和已确认版本的 Gensokyo 实际事件分别验证：

- 入群与离群事件字段解析；
- CQ 段顺序和原始事件段保留；
- 机器人自身事件跳过；
- 主动离群与被移除不作猜测；
- 成员资料查询失败；
- 头像获取失败；
- Markdown 构建失败；
- 发送失败不导致事件处理器崩溃。

## 后续计划

在完成事件样本确认后，再决定是否增加配置类、模板文件、群级开关、Markdown 输出和自动化事件测试。本轮不做未经版本验证的大规模重构。

## 维护边界

- 不接入 qbind；
- 不判断主动退群或被踢；
- 不把 Release006/007/008 的行为差异互相替代；
- 不把头像、群资料或消息正文写入业务数据库。
